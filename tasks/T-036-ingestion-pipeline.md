## Мета

Pipeline для завантаження SOP/manual документів у Azure Blob Storage → автоматичне chunking та embedding → Azure AI Search indexes.

---

## Flow

```
Upload .md/.pdf/.docx to Blob Storage (5 separate containers per source type)
    │
    ▼  (separate blob trigger per container)
┌────────────────────────────────────────────────────────────┐
│  ingest_sop_document      blob-sop/      → `idx-sop-documents`     │
│  ingest_equipment_manual  blob-manuals/  → `idx-equipment-manuals` │
│  ingest_gmp_policy        blob-gmp/      → `idx-gmp-policies`      │
│  ingest_bpr_document      blob-bpr/      → `idx-bpr-documents`     │
│  ingest_history_document  blob-history/  → `idx-incident-history`  │
└────────────────────────────────────────────────────────────┘
    │
    ├── Parse document (markdown/PDF/DOCX → plain text)
    ├── Split into chunks (strategy per container — see below)
    ├── Generate embeddings (Azure OpenAI text-embedding-3-small)
    └── Upsert to corresponding AI Search index
```

### Chunking strategy per container

| Container | Strategy | Reason |
|---|---|---|
| `blob-sop` | 500 tokens, 50 overlap | Standard linear text |
| `blob-manuals` | 500 tokens + equipment_id tag | Equipment ID extracted from filename |
| `blob-gmp` | 500 tokens + clause number metadata | Regulation clause structure |
| `blob-bpr` | **Table-aware**: Markdown tables kept intact (\u2264 1200 tokens); other text 500 tok | CPP tables must not be split — half-tables cause AI to fabricate process limits |
| `blob-history` | Pre-chunked by script; 1 chunk per incident section | Generated by `generate_history_chunks.py` |
